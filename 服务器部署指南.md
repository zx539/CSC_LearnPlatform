# 个性化学习多智能体系统服务器部署指南

本文给出一套可直接落地的 Linux 服务器部署流程（`Python + Gunicorn + Nginx + systemd`）。

## 1. 部署目标与默认信息

- 项目启动入口（开发模式）：`python3 run.py`（默认 `8000` 端口）
- 生产推荐启动方式：`gunicorn backend.app:app`
- 代码目录示例：`/opt/personalized-learning-system`
- 对外域名示例：`learn.example.com`

> 建议使用 Ubuntu 22.04+ 或 Debian 12+。

---

## 2. 服务器初始化

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip nginx git
```

（可选）创建专用系统用户：

```bash
sudo useradd -m -s /bin/bash appuser
sudo su - appuser
```

---

## 3. 拉取项目并安装依赖

```bash
cd /opt
sudo git clone <你的仓库地址> personalized-learning-system
sudo chown -R $USER:$USER /opt/personalized-learning-system
cd /opt/personalized-learning-system

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
pip install gunicorn
```

---

## 4. 配置模型 API 与应用环境变量（星火 + 豆包）

项目读取星火配置有两种方式，推荐使用**环境变量**（生产更安全）。

双模型（Ultra/Lite）建议分别配置：

- `SPARK_ULTRA_API_URL`
- `SPARK_ULTRA_API_AUTH`
- `SPARK_LITE_API_URL`
- `SPARK_LITE_API_AUTH`
- `DOUBAO_API_URL`
- `DOUBAO_API_AUTH` 或 `DOUBAO_API_KEY`

兼容变量（单密钥场景）：

- `SPARK_API_URL`
- `SPARK_API_AUTH`（可直接填密钥，程序会自动补 `Bearer` 前缀）

同时建议配置：

- `APP_SECRET_KEY`（必须替换默认值）
- （可选）`SPARK_CONNECT_TIMEOUT`、`SPARK_READ_TIMEOUT`、`SPARK_MAX_RETRIES`
- （可选）`SPARK_MAX_PARALLEL`、`RESOURCE_MAX_PARALLEL`、`FORM_MAX_PARALLEL`

### 4.4 方式C（推荐生产）：写入 systemd 环境文件

创建仅 root 可读的环境文件：

```bash
sudo tee /etc/personalized-learning.env > /dev/null << 'EOF'
APP_SECRET_KEY=替换为你刚生成的随机密钥
USER_DATA_DIR=/var/lib/personalized-learning/users
SPARK_ULTRA_API_URL=https://spark-api-open.xf-yun.com/v1/chat/completions
SPARK_ULTRA_API_AUTH=Bearer 你的Ultra模型密钥
SPARK_LITE_API_URL=https://spark-api-open.xf-yun.com/v1/chat/completions
SPARK_LITE_API_AUTH=Bearer 你的Lite模型密钥
DOUBAO_API_URL=https://ark.cn-beijing.volces.com/api/v3
DOUBAO_API_KEY=你的豆包密钥
SPARK_CONNECT_TIMEOUT=10
SPARK_READ_TIMEOUT=240
SPARK_MAX_RETRIES=2
SPARK_MAX_PARALLEL=5
RESOURCE_MAX_PARALLEL=3
FORM_MAX_PARALLEL=2
EOF

sudo chmod 600 /etc/personalized-learning.env
```

然后在 systemd 服务中引用：

```bash
sudo sed -i '/^Environment="/d' /etc/systemd/system/personalized-learning.service
sudo sed -i '/^WorkingDirectory=/a EnvironmentFile=/etc/personalized-learning.env' /etc/systemd/system/personalized-learning.service

sudo systemctl daemon-reload
sudo systemctl restart personalized-learning
sudo systemctl status personalized-learning --no-pager
```

## 5. 使用 systemd 托管 Gunicorn

创建服务文件：

```bash
sudo tee /etc/systemd/system/personalized-learning.service > /dev/null << 'EOF'
[Unit]
Description=Personalized Learning Multi-Agent Web App
After=network.target

[Service]
User=appuser
Group=www-data
WorkingDirectory=/opt/personalized-learning-system
EnvironmentFile=/etc/personalized-learning.env
ExecStart=/opt/personalized-learning-system/.venv/bin/gunicorn \
  --workers 2 \
  --threads 4 \
  --timeout 300 \
  --bind 127.0.0.1:8000 \
  backend.app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF







sudo tee /etc/systemd/system/personalized-learning.service > /dev/null << 'EOF'
[Unit]
Description=Personalized Learning Multi-Agent Web App
After=network.target

[Service]
User=ubuntu
Group=ubuntu

    --timeout 300 \
    --bind 0.0.0.0:8000 \
    backend.app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

加载并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable personalized-learning
sudo systemctl start personalized-learning
sudo systemctl status personalized-learning --no-pager
```

## 6. 配置 Nginx 反向代理

创建站点配置：

```bash
sudo tee /etc/nginx/sites-available/personalized-learning > /dev/null << 'EOF'
server {
    listen 80;
    server_name learn.example.com;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300;
    }
}
EOF




# 1. 创建配置文件
sudo tee /etc/nginx/sites-available/personalized-learning > /dev/null << 'EOF'
server {
    listen 80;
    server_name www.example.sx.cn;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

# 2. 启用配置
sudo ln -s /etc/nginx/sites-available/personalized-learning /etc/nginx/sites-enabled/

# 3. 测试配置是否正确
sudo nginx -t

# 4. 重载 Nginx 配置（不中断服务）
sudo systemctl reload nginx
```

启用并重载：

```bash
sudo ln -sf /etc/nginx/sites-available/personalized-learning /etc/nginx/sites-enabled/personalized-learning
sudo nginx -t
sudo systemctl reload nginx
```

## 8. 防火墙与端口建议

- 对外只开放：`80`、`443`
- `8000` 仅监听本机：`127.0.0.1:8000`（已在 Gunicorn 配置中处理）

如使用 UFW：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status
```

## 9. 目录与数据持久化说明

运行中会写入以下目录，请确保服务用户有权限：

- `USER_DATA_DIR` 指向目录（用户数据，建议如 `/var/lib/personalized-learning/users`）
- `outputs/`（每次生成结果）

推荐在服务器上先创建用户数据目录并授权给运行服务的账户（以 `www-data` 为例）：

```bash
sudo mkdir -p /var/lib/personalized-learning/users
sudo chown -R www-data:www-data /var/lib/personalized-learning
sudo chmod 750 /var/lib/personalized-learning/users
```

若你的 `systemd` 服务使用的是其他用户，请将 `www-data:www-data` 替换为对应账号。

建议加入定期备份（如每日 rsync 到备份盘/对象存储）。

## 12. 故障排查清单

1. 页面 502：先看 `systemctl status` 与 `journalctl -u personalized-learning`
2. 登录后接口报错：检查 `APP_SECRET_KEY` 是否配置
3. 模型请求失败：检查 `SPARK_*` / `DOUBAO_*` 环境变量是否正确
4. 静态资源异常：确认 Nginx 已正确反向代理到 Gunicorn
5. 响应慢或超时：调优 `SPARK_*_MAX_PARALLEL`、`RESOURCE_MAX_PARALLEL`、`FORM_MAX_PARALLEL` 与超时参数

## 13. 最小可用校验

部署完成后，访问：

- `https://learn.example.com/login`

完成一次登录并测试：

1. 新建学习项目并触发生成
2. 测试智能辅导接口
3. 查看 `outputs/` 是否产生新 run 文件

至此即可完成生产可用部署。
