# 基于大模型的个性化资源生成与学习多智能体系统（赛题对齐版项目介绍）

## 一、项目概述（面向软件杯 A 组赛题）

本项目围绕“基于大模型的个性化资源生成与学习多智能体系统开发”赛题，面向高校学习场景，构建了一个可运行的 Web 智能学习平台。  
系统以学生自然语言对话为入口，完成“画像构建—资源生成—路径规划—过程评估—智能辅导”的闭环服务，解决传统统一式教学中“资源难匹配、学习路径不清晰、反馈不及时”的痛点。

---

## 二、与赛题功能要求的逐条对齐

| 赛题要求 | 项目实现 |
|---|---|
| 1. 对话式学习画像（不少于 6 维、动态更新） | 通过画像智能体从对话中抽取学习目标、知识基础、认知偏好、薄弱点、学习习惯、阶段状态等多维信息；在学习进度提交后可迭代更新画像 |
| 2. 多智能体协同，至少 5 类个性化资源 | 已实现多智能体协作，生成 6 类资源：讲解文档、知识导图（Mermaid）、练习题、拓展阅读、实操案例、视频学习建议 |
| 3. 个性化学习路径规划与资源推送 | 基于画像 + 学习主题输出阶段化路径（步骤、目标、检查点）；结合学习记录给出针对性资源建议 |
| 4. 智能辅导（可选加分） | 提供问答式智能辅导，支持“结论→原理→示例”的结构化解答，并结合学习上下文连续对话 |
| 5. 学习效果评估（可选加分） | 基于进度问卷与测试问卷进行阶段评估，输出风险判断、达成度分析与后续优化建议 |

## 三、核心创新点

### 1. 从“内容生成”升级到“学习过程编排”

不仅生成资料，更通过问卷和评估形成持续迭代的学习闭环，提升个性化服务的持续性与可执行性。

### 2. 问卷鲁棒性纠偏机制

针对大模型输出结构漂移问题，构建“后端修复 + 前端归一化”双层机制（如题型自动修复），保证问卷可渲染、可提交、可评估。

### 3. 多模型可插拔与工程化调用

统一封装星火与豆包模型调用，支持并发控制、超时与重试，平衡生成质量、响应效率与系统稳定性。

### 4. 异步生成任务机制（本版本重点优化）

针对豆包等模型在复杂任务中耗时长、同步接口易触发 504 的问题，系统将学习方案生成升级为“提交任务 + 状态轮询”模式：

- 前端提交生成请求后立即获得 `task_id`，不再阻塞等待整条链路完成；
- 后端后台执行“画像→资源→路径→问卷”等完整流程；
- 前端通过任务状态接口轮询进度并在完成后一次性加载结果；
- 用户停止等待时，后台任务可继续运行，结果仍可在“已有学习项目”中查看。

### 5. 面向真实部署的数据治理

用户数据独立于代码仓库存储（`USER_DATA_DIR`），支持生产环境独立挂载与旧数据迁移，避免版本更新污染用户数据。

### 6. 学习产品化交互设计

支持 Markdown、公式、导图渲染；用户中心支持头像、密码、密保管理；页面信息结构清晰，降低学习与操作负担。

---

## 四、赛题非功能要求响应

1. **界面与交互**：采用清晰分区和卡片化展示，支持 Markdown/导图等学习友好型内容呈现。  
2. **稳定性**：通过结构化输出约束、异常兜底与模板归一化减少模型不稳定输出带来的前端故障。  
3. **安全性**：账号采用哈希存储；头像上传执行类型、大小、文件头签名与路径安全校验。  
4. **响应体验**：核心生成链路升级为异步任务，前端持续显示任务状态，显著降低长请求超时风险。  
5. **开源合规**：项目技术文档中已对关键开源组件进行说明，便于答辩与材料归档。

---

## 五、技术方案简述

- 后端：Flask + 多智能体服务编排（画像、资源、路径、评估、辅导）  
- 前端：原生 HTML/CSS/JS，支持“生成任务提交 + 轮询回收结果”  
- AI 能力：星火 / 豆包模型统一调用层  
- 存储：用户维度 JSON 持久化 + 异步任务状态持久化（`USER_DATA_DIR/_async_tasks`）  
- 安全：密码与密保哈希、头像上传多层校验
