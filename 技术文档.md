# 基于大模型的个性化资源生成与学习多智能体系统——技术文档

## 1. 文档目的

本文档面向开发者、部署人员与评审专家，完整说明系统的技术架构、核心模块、关键流程、数据模型、接口设计、安全策略与扩展方案。

---

## 2. 系统总览

## 2.1 项目目标

构建一个以学习者为中心的智能学习平台，支持：

1. 学习画像生成  
2. 个性化学习资源生成  
3. 学习路径规划  
4. 过程问卷与评估  
5. 智能辅导答疑  
6. 用户中心与账号安全管理

## 2.2 技术栈

| 层级 | 技术 |
|---|---|
| 后端 | Python 3 + Flask |
| 前端 | HTML + CSS + 原生 JavaScript |
| AI 调用 | 星火（Ultra/Lite）、豆包（可配置） |
| 内容渲染 | Markdown（marked）、KaTeX、Mermaid、highlight.js |
| 数据存储 | 文件系统 JSON（按用户隔离） |

## 2.3 目录结构（核心）

```text
backend/
  app.py                    # Flask 路由入口
  services/
    multi_agent.py          # 多智能体编排与问卷/评估逻辑
    spark_client.py         # 大模型客户端封装
    user_store.py           # 用户数据存储与账号能力
frontend/
  templates/
    index.html              # 主界面
    login.html              # 登录页
    forgot_password.html    # 忘记密码页
  static/
    js/app.js               # 前端核心逻辑
    css/style.css           # 样式
data/                       # 系统样例数据
outputs/                    # 运行输出（可配置）
```

## 3. 总体架构设计

## 3.1 架构分层

1. **表现层（Frontend）**  
   负责用户输入、内容渲染、问卷交互、用户中心弹窗、头像上传与状态管理。

2. **应用层（Flask API）**  
   负责路由鉴权、参数校验、流程编排、结果组织与错误返回。

3. **智能服务层（MultiAgent + SparkClient）**  
   负责调用大模型完成画像/资源/路径/问卷/评估/辅导等任务。

4. **数据层（UserStore + 文件系统）**  
   负责用户账号、学习记录、问卷模板、历史报告、头像文件等持久化。

## 3.2 关键设计原则

- **结构化输出优先**：模型输出尽量约束为 JSON，便于程序消费。  
- **双层纠偏**：后端生成时纠偏 + 前端渲染前归一化，提升稳定性。  
- **用户数据隔离**：用户数据与代码仓库分离，支持部署升级。  
- **可插拔模型**：模型供应商与具体模型可替换，保留统一调用接口。  
- **安全默认值**：账号和文件上传采用严格校验策略。

## 4. 后端模块详解

## 4.1 `backend/app.py`（API 网关）

主要职责：

- 用户认证：注册、登录、忘记密码、修改密码、密保维护
- 学习主流程：生成学习报告、提交学习进度、提交测试问卷
- 用户中心：读取学习项目统计、头像上传与获取
- 渲染辅助：问卷模板标准化、学习画像核心字段提取

### 4.1.1 核心能力点

1. **问卷模板标准化**  
   - 对历史与实时问卷模板执行归一化  
   - 保证问题类型、选项结构、量表字段可用

2. **画像核心提取**  
   - 兼容嵌套/非标准结构画像数据  
   - 保障后续评估输入稳定

3. **用户中心接口**  
   - 输出学习项目数量及项目名列表  
   - 与前端弹窗交互解耦

## 4.2 `backend/services/user_store.py`（用户存储与账号安全）

主要职责：

- 用户注册与登录（密码哈希）
- 密保问题保存与校验（答案哈希）
- 忘记密码重置、登录态改密
- 用户中心统计信息聚合
- 头像保存与读取
- 用户数据目录迁移与隔离

### 4.2.1 数据隔离与迁移

- 使用 `USER_DATA_DIR` 配置用户数据根目录  
- 默认目录可设为仓库外路径（如 `../csc_learnplatform_user_data`）  
- 兼容旧目录 `data/users`，首次访问时可迁移，平滑升级

### 4.2.2 头像上传安全校验

后端进行多层防护：

1. Data URL 格式校验  
2. Base64 大小限制  
3. 解码后字节大小限制  
4. 文件头签名（魔数）校验  
5. 文件名白名单与路径安全控制

## 4.3 `backend/services/multi_agent.py`（多智能体编排）

主要职责：

- 学习画像构建
- 资源生成（多类型）
- 学习路径规划
- 进度问卷/测试问卷生成
- 学习评估
- 辅导答疑

### 4.3.1 问卷类型修复机制

针对模型可能出现的题型漂移（例如本应 `multi_choice` 却返回 `text`），系统在问卷生成后执行 `_repair_question_type`，自动纠偏并补齐必要字段，避免前端渲染异常。

## 4.4 `backend/services/spark_client.py`（模型客户端）

主要职责：

- 多模型配置读取（环境变量）
- 请求封装与接口兼容
- 并发控制（信号量）
- 超时与重试策略
- 响应内容提取

### 4.4.1 工程价值

将模型调用能力从业务逻辑中抽离，降低耦合，便于未来接入更多模型供应商。

## 5. 前端模块详解

## 5.1 `frontend/templates/index.html`

核心界面包括：

- 输入区与学习结果展示区
- 左侧学习标签导航（学习画像、学习资源、学习路径、学习评估、辅导等）
- 顶部用户区（头像 + 用户名）
- 用户中心弹窗（非侧栏页签）
- 密码与密保设置弹窗

## 5.2 `frontend/static/js/app.js`

主要职责：

- 全局状态维护（当前用户、当前学习项目、问卷状态等）
- 与后端 API 的交互封装
- Markdown/公式/导图渲染控制
- 用户中心弹窗交互（打开、加载、更新）
- 头像上传预检与提交
- 问卷模板前端归一化（兜底）

## 5.3 前端交互关键点

1. 登录页“忘记密码”入口采用右下角小字，弱化主流程干扰。  
2. 用户中心不占用主导航页签，改为“点击头像进入”，减少界面负担。  
3. 密保修改和密码修改通过弹窗进行，交互更聚焦。  
4. 用户中心展示学习项目数量和项目名，形成学习资产视图。

## 6. 数据模型与存储规范

## 6.1 用户数据组织

每个用户一个独立目录，典型内容包括：

- `profile.json`：账号及基础信息
- `security_questions.json`：密保问题（答案哈希）
- `learning_runs/*.json`：学习项目运行数据
- `avatar.*`：头像文件

## 6.2 学习项目（Run）关键字段

- `run_name`：项目名  
- `topic`：学习主题  
- `profile`：学习画像  
- `resources`：学习资源集合  
- `path`：学习路径  
- `progress_form_template`：进度问卷模板  
- `test_form_template`：测试问卷模板  
- `history`：学习进度与评估历史

## 6.3 输出目录

`outputs/` 用于运行期输出与可视化产物存放，部署时可单独授权。

## 7. 关键流程（时序说明）

## 7.1 学习项目生成流程

1. 用户输入学习目标与背景  
2. 后端调用画像智能体生成 profile  
3. 后端调用资源智能体生成多类型资源  
4. 后端调用路径智能体生成阶段计划  
5. 生成初始进度问卷模板并持久化  
6. 前端渲染完整学习报告

## 7.2 学习进度反馈流程

1. 前端提交进度问卷  
2. 后端标准化问卷模板与回答数据  
3. 评估智能体输出阶段结论、风险与建议  
4. 返回前端展示并写入历史

## 7.3 忘记密码流程

1. 用户输入用户名获取密保问题  
2. 用户提交三问答案 + 新密码  
3. 后端校验答案哈希  
4. 校验通过后重置密码哈希并持久化

## 7.4 头像上传流程

1. 前端选择图片并做类型/体积初检  
2. 转 Data URL 发送到后端  
3. 后端执行格式、大小、签名与路径校验  
4. 存储到用户目录并返回可访问地址  
5. 前端刷新头像显示

## 8. API 设计概览

> 说明：以下为核心接口分组，具体参数以代码实现为准。

| 分组 | 代表接口 |
|---|---|
| 认证 | `/api/auth/register` `/api/auth/login` `/api/auth/security-questions` `/api/auth/reset-password` |
| 用户中心 | `/api/user/center` `/api/user/change-password` `/api/user/security-questions` |
| 头像 | `/api/user/avatar`（上传/获取） |
| 学习主流程 | `/api/generate` `/api/progress/checkin` `/api/user/run/<run_name>` |
| 辅导 | `/api/tutor/chat`（按实现命名可能略有差异） |

## 9. 配置与部署要点

## 9.1 环境变量（示例）

- 模型配置：`SPARK_*`、`DOUBAO_*`（按实际 `.env` 约定）  
- 数据目录：`USER_DATA_DIR`  
- 运行环境：Flask 基础配置项

## 9.2 用户数据目录建议

生产环境建议将用户数据目录放在仓库外，例如：

```bash
sudo mkdir -p /opt/csc_learnplatform_user_data /opt/csc_learnplatform_outputs
sudo chown -R <部署用户>:<部署用户组> /opt/csc_learnplatform_user_data /opt/csc_learnplatform_outputs
chmod -R 750 /opt/csc_learnplatform_user_data /opt/csc_learnplatform_outputs
```

并通过 `USER_DATA_DIR=/opt/csc_learnplatform_user_data` 指向该目录。

## 10. 安全设计

## 10.1 账号安全

- 密码采用哈希存储，不保存明文  
- 密保答案采用哈希校验  
- 改密流程支持旧密码校验与密保校验两种路径

## 10.2 文件上传安全

- 前端白名单限制（MIME + 体积）  
- 后端二次严格校验（签名、大小、文件名、路径）  
- 杜绝任意文件写入与路径穿越

## 10.3 数据安全与仓库治理

- `data/users/` 忽略提交，避免测试/生产用户数据入库  
- `.venv/`、`venv/`、`env/` 忽略提交，避免环境目录污染仓库

## 11. 稳定性与可维护性

## 11.1 稳定性机制

- 模型请求超时与重试  
- 问卷模板双层标准化  
- 历史数据兼容处理  
- 关键字段提取容错

## 11.2 可维护性设计

- 路由层、服务层、存储层职责分离  
- 模型客户端独立封装  
- 用户能力统一收口在 `UserStore`  
- 前端集中在 `app.js` 进行状态与交互管理

---

## 12. 性能与扩展建议

## 12.1 性能优化方向

1. 增加常见生成结果缓存（按主题与难度粒度）  
2. 为问卷与评估结果引入异步任务队列  
3. 对学习历史页面启用分页与按需加载

## 12.2 扩展方向

1. 引入关系型数据库替代部分 JSON 文件存储  
2. 加入 RBAC（管理员/教师/学生）权限体系  
3. 接入学习行为埋点与效果分析看板  
4. 支持多语言学习资源生成与跨学科推荐

---

## 13. 已知边界与后续计划

- 当前存储以文件系统为主，适合中小规模部署；大规模并发建议数据库化。  
- 智能体输出质量受模型与提示词影响，仍需持续优化提示词与评估规则。  
- 未来计划引入自动化测试集（接口、前端交互、问卷一致性）提高回归效率。

---

## 14. 结语

本系统在技术上实现了“多智能体协同 + 学习闭环 + 工程化部署”的统一落地。  
其价值不仅在于内容生成，更在于持续驱动学习过程、反馈和优化，是面向真实教学场景的可用型智能学习平台。
